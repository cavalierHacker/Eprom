Welcome.

Thank you for purchasing our theme. Before starting, be sure to have a code editor (we recommend Sublime Text 2) and some working knowledge of HTML and CSS.
So you can fully enjoy this template you will need a real web server (online or local) to work properly.

Basics.

Section body Section contains the class with the name of current page. You are using this class to identify the current page and class name contains the prefix "page-". All the pages have been divided into five "main" sections: #header, #page-header, #main-content, #footer-top, #footer. .container determines the page width and also centralize it. You can adjust this width from css/style.css file.

Simplified example of fullwidth page structure:

<!-- header -->
<header id="header">
    <div class="container">
      [HEADER_CONTENT_HERE]
  </div>
</header>
<!-- /header -->

<!-- page header -->
<section id="page-header">
  <div class="container clearfix">
    [PAGE_HEADER_CONTENT_HERE]
  </div>
</section>
<!-- /page header -->

<!-- content -->
<section id="main-content" class="container clearfix">
  [CONTENT_HERE]
</section>
<!-- /content -->

<!-- footer-top -->
<section id="footer-top">
  <div class="container clearfix">
    [FOOTER_TOP_CONTENT_HERE]
  </div>
</section>
<!-- /footer-top -->

<!-- footer -->
<footer id="footer" class="clearfix">
  <section class="container clearfix">
    [FOOTER_CONTENT_HERE]
  </section>
</footer>
<!-- /footer -->

Header.

Logo.

If the logo height is more than 86px, you may need to change #header and #logo height settings from the css/style.css file.

Main navigation.

Navigation is just basic unordered list and based on this list JS script generates responsive navigation element select.responsive-nav and places in nav#main-nav. And .current obviously indicates the active page.

Page header.

You can place your page title, subtitle, share buttons, masonry boxes and much more. Check all pages and see the ability to add elements in this section.

A simple example.

<!-- page header -->
<section id="page-header">
  <div class="container clearfix">
    <hgroup>
      <h1 class="page-title">Elements</h1>
      <h2 class="page-subtitle">Lorem ipsum dolor sit amet.</h2>
    </hgroup>
    <!-- share buttons -->
    <div id="share-wrap">
      <div id="share" data-url="http://CURRENT_PAGE_URL" data-text="CURRENT_PAGE_TITLE." data-title="share"></div>
    </div>
    <!-- /share buttons -->
    <hr>
  </div>
</section>
<!-- /page header -->

Please note attribute data-url must have the full address of the current page.

Page header can contain a unique background. If you want to create your own background, open the file css/style.css and go to "Page header", add a CSS rule 

.page-YOUR-UNIQUE-PAGE-NAME #page-header { background: #000 url(../placeholders/blog-bg.jpg) center center no-repeat; }

Please note background is not available on mobile devices. You can change this in css/media-queries.css but keep in mind that the backgrounds are large and may slow down the loading of the page.

Masonry boxes.

In the main content or in page header you can put the masonry boxes grid. In the boxes can be placed images, videos, sliders, and text. Each box must have a class defining the height and width. Below is a list of available classes.

  1. .width-1-4 - Width 233px (1/4 of fullwidth).
  2. .height-1-4 - Height 233px.
  3. .width-1-2 - Width 468px (1/2 of fullwidth).
  4. .height-1-2 - Height 468px.
  5. .width-3-4 - Width 703px (3/4 of fullwidth).
  6. .height-3-4 - Height 703px.
  7. .media - Special class for media embeds.
  8. .text-box - Text box class (example included in artist.html file).

For example:

<!-- masonry boxes -->
<div class="masonry-wrap clearfix">
  <!-- 1/2 -->
  <div class="masonry width-1-2 height-1-2">
    
    [CONTENT]

  </div>
  <!-- /1/2 -->

  <!-- 1/4 -->
  <div class="masonry width-1-4 height-1-4">
    
    [CONTENT]

  </div>
  <!-- /1/4 -->

  <!-- 1/4 -->
  <div class="masonry width-1-4 height-1-4">

    [CONTENT]

  </div>
  <!-- /1/4 -->

  <!-- 1/2 width and 1/4 height -->
  <div class="masonry width-1-2 height-1-4">

    [CONTENT]

  </div>
  <!-- /1/2 width and 1/4 height -->

</div>
<!-- /masonry boxes -->

You can check sample boxes from gallery.html file.

Portfolio.

Portfolio filter.

All projects are placed in <section class=".items .portfolio"> and every project is a single <article>. Every project has data-categories and data-tags attributes which will be used in filtering them. A single project can has multiple categories and tags and they need to be separated with space. There isn't any naming rules but you should use only lowercase and single word (use dash to replace spaces) names to avoid any possible issues. And you don't need to use class .last with portfolio items, like usually with columns.

Basic example:

<!-- item -->
<article class="col-1-4" data-categories="[CATEGORIES]" data-tags="[TAGS]">

  <!-- image -->
  <a href="[PROJECT_LINK]"><img src="[IMAGE]" alt="" title="" /></a>

  <!-- item footer -->
  <footer>
    <h2><a href="[PROJECT_LINK]"> [TITLE] </a></h2>
    <div class="cat"> [CATEGORIES] </div>
  </footer>
  <!-- /item footer -->
  
</article>
<!-- /item -->

You can use various effects for portfolio images, details can be found in the "Elements" section or open /elements.html file.

Example of category filter:

<!-- cat filter -->
<ul id="cat-filter">
  
  <li><a data-categories="*">All</a></li>
  <li><a data-categories="new">New Tracks</a></li>
  <li><a data-categories="free">Free!</a></li>
  [...]

</ul>
<!-- /cat filter -->

Example of tag filter:

<!-- tags filter -->
<ul id="tag-filter">

  <li><a data-tags="*">All Artists</a></li>
  <li><a data-tags="dj-envato">DJ Envato</a></li>
  <li><a data-tags="dj-graphicriver">DJ Graphicriver</a></li>
  [...]
  
</ul>
<!-- /tags filter -->

Star selector (*) is used to select all items. Category filter is located in #page-header and tag filter is in #main-content section.

Conatct.

Setting up the contact form is really simple task. All you need to do is to open plugins/config.php file and replace $your_email_address value with your own one and that's it!

PHP Plugins.

Just like with the contact form, all the php scripts settings can be be changed from the plugins/config.php file. All settings should be pretty self-explanatory. Twitter integration.

Step 1: Sign in to Twitter.

Sign in to Twitter. If you don’t already have a Twitter account, you can sign up at the Twitter home page to create one.

Step 2: Create your Twitter Application.

a. After you sign in, go to application creation page. Be sure to fill in your application’s name, a description about your application and your site URL in the WebSite field. In the Callback URL field, leave empty. This tells your app to return to this location after authenticating.

b. Read and agree to Twitter’s "Developer Rules Of The Road". Check the box next to "Yes, I agree".

c. In the space below, enter the characters you see in the CAPTCHA and click on the "Create your Twitter application" button.

d. If the form is filled well you should see your Twitter app’s detail page. Now you need to generate your access token.
Click on the "Create my access token".

e. Your tokens should be generated if they are not visible on the page, wait a moment and click on the "Details" tab again.

f. Now you have the data needed to properly display the tweets in your template.

To display tweets in this theme you need:

  1. Twitter account.
  2. Twitter application.
  3. Twitter username.
  4. Consumer key.
  5. Consumer secret.
  6. Access token.
  7. Access token secret.

Javascripts.

All Javascript files are located in /js folder. By default theme uses minified version of the all scripts to reduce page loading times and to save your site bandwidth. All used JavaScript scripts can be found in HEAD section of every page.

  1. jquery-1.7.1.min.js - JavaScript Library.
  2. selectivizr-and-extra-selectors-min.js - CSS3 pseudo-class and attribute selectors for IE 6-8.
  3. respond.min.js - A fast & lightweight polyfill for min/max-width CSS3 Media Queries.
  4. jquery.easing-1.3.min.js - More easing options for sliders.
